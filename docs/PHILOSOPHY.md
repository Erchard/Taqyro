# Taqyro — Project Philosophy

### Purpose of This Document

This document describes the philosophy behind **Taqyro**.

It explains what the product is, what it is not, what experience it should create, and what principles should guide product, design, engineering, and content decisions.

Taqyro is not just a question app.  
It is not a dating app.  
It is not a social network.  
It is not a personality test.

Taqyro is a product about **choice, boundaries, shared values, and offline connection**.

---

## 1. Core Belief

The core belief of Taqyro is simple:

**People reveal themselves through important choices.**

Not through profile descriptions.  
Not through photos.  
Not through polished bios.  
Not through status markers.  
Not through the identity labels they choose for themselves.

A person’s deeper compatibility with others often becomes visible when they answer questions like:

- What is acceptable?
- What is unacceptable?
- What is negotiable?
- What is non-negotiable?
- What would you sacrifice?
- What would you refuse to sacrifice?
- Where is your boundary?
- What kind of people could you actually cooperate with?

Taqyro exists to make those choices visible in a simple, direct, non-manipulative way.

---

## 2. Choice Before Identity

Most social products start with identity.

They ask the user to define themselves:

- name;
- photo;
- age;
- gender;
- bio;
- job;
- education;
- hobbies;
- interests;
- location;
- lifestyle;
- preferences.

Taqyro starts differently.

It asks a question.

The user is not asked to describe who they are.  
The user is asked to make a choice.

This is the central philosophical difference.

In Taqyro, identity is not the starting point.  
**Choice is the starting point.**

The product should always protect this idea.

---

## 3. The First Screen Matters

The first screen should express the whole product.

It should not be:

- a registration form;
- a welcome slideshow;
- a marketing explanation;
- a profile setup screen;
- a request for personal data;
- a list of permissions;
- a menu.

The first screen should be a meaningful question.

The user should immediately feel:

> “This app is not asking me to perform an identity. It is asking me to choose.”

That first question must create curiosity, tension, and involvement.

The first experience should happen before the user gives anything to the product except an answer.

---

## 4. No Value Before Registration Is a Mistake

Taqyro should not ask for registration before delivering value.

The product must earn the right to ask the user to save their result.

The correct sequence is:

```text
Question
→ Answer
→ Feedback
→ More answers
→ Circle starts forming
→ User sees value
→ Then save/account prompt
```

The wrong sequence is:

```text
Register
→ Create profile
→ Configure settings
→ Answer questions
→ Maybe see value later
```

Taqyro should avoid the second sequence.

The user should feel the product before they commit to it.

---

## 5. The Product Is Not a Personality Test

Taqyro must not become a personality test.

It should not assign types, labels, or psychological interpretations.

It should not say:

* “You are a libertarian.”
* “You are a collectivist.”
* “You are a traditionalist.”
* “You are a rebel.”
* “You are a protector.”
* “You are type X.”
* “You belong to archetype Y.”

These labels may feel entertaining, but they distort the product.

Taqyro should not interpret the user.
Taqyro should compare choices.

The product should say:

```text
126 people answered the same way as you on the questions that matter to you.
```

Not:

```text
You are a freedom-oriented principled individualist.
```

This distinction is essential.

---

## 6. Facts Over Interpretation

Taqyro should be factual.

The app should avoid unnecessary meaning-making.

It should not pretend to know the user’s soul.
It should not explain the user to themselves.
It should not create pseudo-psychological conclusions.

The product only knows:

* which questions were shown;
* how the user answered;
* which answers were used as filters;
* how many people match those answers;
* which people are in the resulting circle.

That is enough.

The power of the product comes from the fact that the user can see the result of their own choices.

The app does not need to decorate that with invented personality language.

---

## 7. “Not Important” Is a Philosophical Feature

The answer **Not important** is not just a UX option.

It is a philosophical principle.

It means:

> “This issue does not define who belongs in my circle.”

Taqyro should not force users to treat every question as important.

People have different boundaries.

For one person, a question may be a red line.
For another, the same question may be irrelevant.

The product must respect that.

Therefore:

```text
Yes = filter by Yes
No = filter by No
Not important = no filter
```

If the user chooses **Not important**, the question is ignored for matching.

It does not matter how others answered it.
It does not matter whether others answered it at all.

For this user, this question is not part of the boundary.

This is one of the most important rules in the product.

---

## 8. The Circle Is Built From Boundaries

Taqyro is not matching people by everything they answered.

It is matching people by what the user treats as important.

A user’s circle is not defined by all their opinions.

It is defined by their meaningful boundaries.

This matters because people do not need perfect sameness.

They need alignment on what matters.

Two people can disagree on many things and still cooperate if those things are not central to them.

But one disagreement on a principle that matters deeply can make cooperation impossible.

Taqyro’s matching logic should reflect this.

---

## 9. The Product Should Not Seek Total Similarity

Taqyro should not try to find a perfect clone of the user.

The goal is not:

> “Find someone who thinks exactly like me about everything.”

The goal is:

> “Find people who align with me on the questions that are important to me.”

This difference is crucial.

A healthy circle should allow difference where difference is acceptable.
It should only filter where the user has made something important.

That is why **Not important** is not a weak answer.
It is an explicit decision not to use that question as a boundary.

---

## 10. The Circle Should Feel Earned

The user should feel that the circle was not randomly generated.

It was formed through their own choices.

Every meaningful answer should have a visible effect:

```text
Your circle: 12,400 people
Your circle: 3,800 people
Your circle: 740 people
Your circle: 126 people
```

This visible narrowing creates the central emotional loop:

> “My choices are shaping the people I might meet.”

The circle is not a recommendation feed.
The circle is the visible consequence of the user’s decisions.

---

## 11. The Experience Should Feel Like a Game, Not a Survey

Taqyro asks questions, but it should not feel like a questionnaire.

A questionnaire feels like work.
A game feels like discovery.

The difference is feedback.

After each answer, the user should see something change.

The user should not feel:

> “I am filling out a form.”

The user should feel:

> “I am narrowing the world.”

This is the product experience.

The loop should be fast:

```text
Read
→ Choose
→ See result
→ Continue
```

No long explanations.
No heavy screens.
No complex settings during the first flow.

---

## 12. Curiosity Is the Engine

The user should want to answer the next question.

Not because the app forces them.

Because they want to know:

* How many people will remain?
* Is my answer common or rare?
* Will this question split the circle?
* Am I close to a useful circle?
* What happens if I continue?
* Are there people nearby who answered like me?

The product should create curiosity through consequences.

Each answer should feel like a move in a game.

---

## 13. The Questions Are the Product

In Taqyro, questions are not just content.

Questions are the core product material.

A weak question makes the product weak.
A strong question makes the product alive.

Good questions should be:

* short;
* sharp;
* understandable;
* emotionally engaging;
* morally or socially meaningful;
* answerable in one tap;
* not trivial;
* not purely factual;
* not obviously one-sided;
* capable of splitting people.

A good Taqyro question should make the user pause for a second.

Not because it is confusing.
Because it touches something real.

---

## 14. Questions Should Not Be Cheap Outrage

Sharp questions are not the same as inflammatory questions.

Taqyro should not chase outrage.

The goal is not to provoke anger.
The goal is to reveal boundaries.

A question should be strong, but not manipulative.

Bad question style:

```text
Do you also hate people who believe X?
```

Good question style:

```text
Could you cooperate with someone who believes X?
```

The first creates tribal aggression.
The second reveals a boundary.

Taqyro should prefer the second.

---

## 15. Universal First, Local Later

The first questions should be globally understandable.

They should not depend on one country’s politics.

The first experience should work anywhere.

Early questions should focus on universal tensions:

* truth vs loyalty;
* privacy vs safety;
* mercy vs punishment;
* law vs family;
* freedom vs control;
* peace vs dignity;
* equality vs liberty;
* forgiveness vs revenge;
* tradition vs change;
* individual choice vs collective good.

Local and political questions can exist later.

For example, Ukraine-specific questions may be important for Ukrainian users.

But the first screen should be universal.

The product should first reveal how the app works, then allow deeper local filtering.

---

## 16. No Endless Feed

Taqyro should not become another endless feed.

An endless feed optimizes for scrolling.
Taqyro should optimize for movement from choice to circle to action.

The product should avoid design patterns that turn people into content.

The user should not be encouraged to browse humans endlessly.

The circle is not meant to become a marketplace of profiles.

It is meant to become a doorway to connection.

---

## 17. Profiles Are Secondary

Profiles are not the heart of Taqyro.

A profile can exist later, but it should be minimal and functional.

The product should not rely on:

* profile photos;
* attractive bios;
* lifestyle branding;
* social status;
* popularity metrics;
* follower counts;
* likes.

If identity is needed, it should serve trust and coordination, not vanity.

The core unit of meaning is not the profile.

The core unit is the answer.

---

## 18. People Are Not Content

Taqyro should avoid turning people into swipeable content.

This is especially important because the app deals with values, boundaries, and potential offline connection.

People should not feel like cards in a marketplace.

Initial people views can be anonymous or semi-anonymous.

The product can show:

* match count;
* shared important answers;
* approximate distance;
* willingness to join a group;
* willingness to act offline.

But it should avoid making appearance and status the first layer.

---

## 19. From Matching to Action

The final purpose of Taqyro is not matching.

Matching is only a bridge.

The real purpose is:

> help people with shared values move toward offline connection and cooperation.

After the circle is found, the product should ask:

```text
What do you want to do with this circle?
```

Possible answers:

* see people nearby;
* form a small group;
* join an action;
* create an action;
* narrow further.

The product should guide users from abstract similarity to concrete interaction.

---

## 20. Offline Cooperation Is the Long-Term Goal

Taqyro should not be optimized only for online engagement.

The deeper goal is offline cooperation.

This can include:

* small group meetings;
* local initiatives;
* volunteering;
* discussions;
* mutual support;
* community projects;
* learning groups;
* coordinated action.

The product should eventually answer:

> “We found people who share important boundaries. What can they do together?”

A successful Taqyro interaction is not just a message sent.

It is a step toward trust, meeting, or shared action.

---

## 21. Small Groups Matter More Than One-to-One Matching

One-to-one matching can be useful.

But the stronger long-term format is small groups.

Small groups reduce pressure.
They are safer than direct one-to-one contact.
They are better suited for offline cooperation.
They can become real local communities.

Taqyro should eventually support groups of 5–7 people formed from a focused circle.

The product should not copy dating app mechanics by default.

It should create formats that fit cooperation, not only attraction or conversation.

---

## 22. Trust Should Be Built Through Action

Trust should not come from likes, followers, or profile polish.

Trust should come from behavior.

Over time, Taqyro can support trust signals such as:

* participated in an action;
* joined a small group;
* showed up;
* was confirmed by others;
* organized something;
* cooperated respectfully.

These signals should be careful, non-toxic, and not turn into public popularity scores.

The goal is not social ranking.

The goal is practical trust.

---

## 23. Privacy Is Structural

Taqyro deals with sensitive information.

Answers may reveal political, moral, religious, or personal boundaries.

Therefore privacy cannot be an afterthought.

The product should avoid exposing:

* exact location;
* full answer history by default;
* sensitive answers to random users;
* unnecessary personal identity;
* private boundaries outside the matching context.

The app should reveal only what is needed for the next step.

The safest default is:

> show matching facts, not full personal exposure.

---

## 24. Approximate Location Only

Because Taqyro may lead to offline connection, location is important.

But exact location is dangerous and unnecessary.

The product should use approximate location:

* same city;
* same area;
* nearby range;
* distance bucket;
* region.

It should not show exact coordinates or home-like precision.

The goal is:

> “There are people from your circle nearby.”

Not:

> “This person is exactly here.”

---

## 25. The Product Should Be Honest About Counts

The circle count must be treated carefully.

If the app says:

```text
126 people match your important answers.
```

That should mean something real.

The product should avoid fake numbers.

If the database is small at the beginning, the product should be honest.

Possible early-stage wording:

```text
In the current network, 12 people match your important answers.
```

Or:

```text
Your answer pattern is rare in the current network.
```

Do not fake scale.

The product can still be engaging with small numbers if the experience is honest.

---

## 26. Early Cold Start Is a Product Problem, Not a Reason to Fake

At the beginning, there may not be enough users.

That is normal.

The product should solve cold start through:

* strong question loop;
* shareable results;
* invite mechanics;
* local launches;
* small communities;
* waitlists by city;
* “invite people to compare answers” flows.

It should not solve cold start by inventing fake people.

Trust is more important than impressive numbers.

---

## 27. The App Should Be Globally Understandable

Even if the first target market is local, the concept should be globally understandable.

The app should not depend on one political context.

The universal layer should work everywhere:

```text
Answer meaningful questions.
See who answered like you.
Find your circle.
Do something together.
```

Local layers can add regional relevance later.

This keeps the product scalable.

---

## 28. The App Should Avoid Ideological Capture

Taqyro can contain political or controversial questions, but the product itself should not become propaganda for one ideology.

It should not tell users what to believe.

It should not reward one answer as correct.

It should not shame users for their choices.

The product is a matching system, not a moral authority.

It should help people find alignment, not enforce a worldview.

---

## 29. Neutrality Does Not Mean Blandness

Taqyro should be neutral as a system, but not bland as an experience.

Questions can be sharp.
Feedback can be emotionally engaging.
The narrowing effect can be dramatic.

But the app should not judge the user.

It can say:

```text
Your circle became much smaller.
```

It should not say:

```text
You chose the brave answer.
```

The app should create intensity without moral scoring.

---

## 30. The User Controls What Matters

The user decides which questions matter.

This is a major philosophical principle.

A question does not become a boundary because the app says it is important.

A question becomes a boundary because the user answers **Yes** or **No**.

If the user answers **Not important**, the app must respect that.

The product should not secretly use ignored questions for matching.

The user’s explicit choice defines the filter.

---

## 31. No Hidden Scoring System

Taqyro should avoid hidden compatibility scores in the early product.

The matching should be explainable.

A user should be able to understand why someone is in their circle:

```text
They answered the same way on your 12 important questions.
```

This is clearer than:

```text
You are 87% compatible.
```

A percentage may be added later, but the foundational logic should remain transparent.

The user should never feel that the app is making mysterious judgments.

---

## 32. Matching Should Be Directional

One subtle but important principle:

A user’s circle is defined by that user’s important questions.

This means matching can be directional.

Example:

* User A says Q1 is important and answers Yes.
* User B also answered Yes, so B can be in A’s circle.
* But User B may have additional important questions that A has not answered.
* Therefore A may be in B’s circle only after satisfying B’s important filters.

This is acceptable.

The product should eventually handle mutual matching carefully.

But philosophically, each person owns their boundaries.

---

## 33. Mutuality Matters Before Contact

Even if someone appears in a user’s circle, contact should not be forced.

Connection should require consent.

The product should support:

* request contact;
* accept/decline;
* optional additional question before contact;
* group opt-in;
* action opt-in.

Shared answers are not permission to contact aggressively.

The product should encourage intentional interaction.

---

## 34. The Product Should Reward Depth, Not Addiction

Taqyro should be engaging, but it should not become exploitative.

The goal is not to maximize time spent mindlessly answering questions.

The goal is to help the user reach a useful circle and then move forward.

The app should not keep asking questions forever just to increase engagement.

Once the circle is useful, the app should clearly offer next steps.

The product loop should have a purpose.

---

## 35. Question Quality Is a Long-Term Asset

The question library is one of the most valuable assets of Taqyro.

It should be curated carefully.

Questions should be reviewed for:

* clarity;
* cultural portability;
* emotional strength;
* fairness;
* non-manipulative wording;
* ability to split users meaningfully;
* safety;
* moderation risk.

Over time, the question system may become the intellectual core of the company.

It deserves high standards.

---

## 36. User-Created Questions Are Important but Dangerous

Users should eventually be able to add their own questions.

This is powerful because users know their own boundaries better than the product team.

But it is risky because user-created questions can become:

* abusive;
* hateful;
* manipulative;
* defamatory;
* doxxing-oriented;
* spammy;
* low quality;
* duplicates.

Therefore custom questions should start as personal or contextual.

They should not automatically enter the global question library.

Public promotion of custom questions requires moderation.

---

## 37. Language Should Be Precise

Words matter in Taqyro.

Avoid language that creates hostility:

* enemies;
* wrong people;
* purge;
* filter out bad people;
* outsiders;
* traitors;
* inferior;
* cleanse;
* destroy.

Prefer language that describes alignment:

* circle;
* important questions;
* shared answers;
* boundaries;
* nearby;
* group;
* action;
* connection;
* cooperation.

The product should help users find alignment without encouraging dehumanization.

---

## 38. “Circle” Is Better Than “Match”

The word “match” often suggests dating apps.

Taqyro may include matching, but the broader concept is a circle.

A circle can include:

* people to talk with;
* people nearby;
* small groups;
* collaborators;
* local communities;
* action participants.

“Circle” also suggests a bounded group, which fits the narrowing mechanic.

Use “match” only where technically necessary.

---

## 39. Action Is the Product’s Moral Direction

The product should not end with:

```text
You found similar people. Scroll them.
```

It should move toward:

```text
You found similar people. Do something together.
```

This is the ethical and strategic difference.

Taqyro should help users transform shared values into shared action.

That is what makes it different from quiz apps, dating apps, and social networks.

---

## 40. The Product Should Stay Simple

The most dangerous product risk is overcomplication.

Taqyro can easily become:

* a survey tool;
* a personality test;
* a dating app;
* a political network;
* a social media feed;
* a meetup platform;
* a group chat app;
* a civic organizing tool.

It may eventually touch all of these areas.

But the core must stay simple:

```text
Question → Answer → Circle → Action
```

Every new feature should be judged by whether it strengthens this path.

If it does not, it should wait.

---

## 41. What Taqyro Is

Taqyro is:

* a choice-first mobile experience;
* a way to find people through important answers;
* a circle-building tool;
* a bridge from values to connection;
* a possible foundation for offline cooperation.

---

## 42. What Taqyro Is Not

Taqyro is not:

* a personality test;
* a dating app;
* a classic social network;
* a political propaganda tool;
* an endless feed;
* a profile-first app;
* a popularity contest;
* a psychological labeling engine;
* a place for outrage farming.

---

## 43. Product North Star

The long-term question for the product is:

> Can we help people find others who align with them on important boundaries and turn that alignment into real-world cooperation?

The MVP question is simpler:

> Will users keep answering meaningful questions because they want to see their circle narrow?

Both questions should guide development.

---

## 44. Final Summary

Taqyro is built on the belief that important choices reveal meaningful alignment.

The app should start with a question, not a profile.

It should show the user how their choices narrow the world into a circle.

It should avoid psychological labels, ideological judgment, and endless scrolling.

It should respect that some questions do not matter to some users.

It should help users move from shared answers to real connection.

The shortest expression of the product philosophy is:

```text
Choice first.
Only important answers matter.
The circle is built by boundaries.
The goal is action, not scrolling.
```
